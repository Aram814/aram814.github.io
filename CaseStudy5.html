Women's Fellowship

Project Summary: The Women’s Fellowship Bulletin is a live website built to replace a vandalized physical bulletin board. The challenge was to create an accessible, easy-to-use digital bulletin for elderly women with minimal-to-average tech experience. My solution was a simple, React-based site hosted on GitHub with Supabase handling backend data. Information was collected in-person, digitized, and then published online. The result is a secure, accessible, and maintainable platform that improves communication and community engagement.

Overview Challenge Research Design Solution Impact

Project Overview

Project Type: Live Website / UX Design

Role: UX Designer & Developer

Tools Used: React, Supabase, GitHub Pages

The Challenge

Problem Statement: The original bulletin board was unreliable (prone to vandalism) and inaccessible to women who weren’t physically present.

Why It Matters: The bulletin was the group’s main communication tool. Without it, members missed important updates and events.

Constraints: Audience included elderly women with limited tech literacy, so design needed to prioritize simplicity and accessibility.

Research & Discovery

Data Gathering: Information was initially collected in person and on paper before being digitized.

Audience Needs:

  • Clear, readable typography
  • Minimal navigation steps
  • Accessible contrast and layout
  • Works across devices, especially basic laptops/phones

Reference: The church’s main website provided context for tone and style but the bulletin was created independently to focus on the fellowship’s needs.

Key Insight: The community valued reliability and clarity over advanced design features.

Design Process

Approach:

  • Started with hand sketches to define layout and flow.
  • Developed in React with hosting on GitHub Pages.
  • Integrated Supabase as a backend database for bulletin updates.

Design Considerations:

  • Accessibility: WCAG-guided choices — large fonts, strong contrast, simple menus.
  • Ease of Use: Minimal clicks to reach announcements, calendar, or resources.
  • Scalability: Designed so fellowship leaders could eventually manage updates themselves.

The Solution

The final deliverable was a live website replacing the unreliable physical bulletin board.

  • Core features: Event listings, announcements, scripture/devotional content, resources.
  • Backend: Supabase database for storing and managing updates.
  • Frontend: Clean React interface, responsive and accessible.
  • Hosting: GitHub Pages for reliability and cost-effectiveness.

Impact & Results

  • Members now have consistent access to fellowship updates regardless of location.
  • Eliminated issues of vandalism and visibility tied to physical boards.
  • Elderly members report being able to navigate the site more easily than expected due to its simplicity.

Next Steps:

  • Gather additional feedback from group members.
  • Provide training sessions so leaders can confidently update content themselves.
  • Continue monitoring Supabase backend and GitHub hosting for maintenance.
  • Explore adding optional print-friendly exports for members who prefer paper.
