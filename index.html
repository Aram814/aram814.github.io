Personal Project: Highscore Tracker

During my coursework for UX Design, I was given the prompt of creating an app that could track users arcade high-scores. The goal was to be able to create an app that users could access and submit their high-scores at arcade in real time. A main focus off the bat was having machine-verified score submission. It could be a game-changer. As the project continued, it became clear that there were more possibilities to bring into this app. What if you could not only have access to viewing verified scores, but also a play card balance, tickets tracker, arcade locations, popular games, add in a social following element, and more!

My Role & The Challenge

As the sole UX Developer, I handled the entire process from user research to final code. The primary challenge involved transforming a manual, often frustrating process into a seamless digital experience that builds trust and enhances player engagement.

My Process

I started with a competitive analysis and conducted informal interviews with 5 potential users to understand their pain points with similar sites. Key insight: users were frustrated by complex checkout processes and poor product filtering.

Leveraging insights from the discovery phase, I defined core user needs and project objectives. This involved creating user personas and journey maps to establish a clear understanding of the problem space and user goals.

I translated defined requirements into tangible designs, starting with wireframes to explore layout and functionality. Iterative prototyping, from low-fidelity to high-fidelity, allowed for testing and refinement of the user interface and interaction flows.

I brought the designs to life using modern front-end technologies, focusing on clean, modular code. This involved building responsive UI components and ensuring the implementation met accessibility standards and performance goals.

Outcome & Learnings

There was a comprehensive understanding of core user need of directly focusing on the app's capability for seamless score submission, real-time leaderboards, and community engagement. A key learning was that building trust through transparent score verification is paramount to user adoption and satisfaction. As I continue, I plan to enhance this app with social aspects and further develop the validation process.

Project Beta: Personal Task Manager App

A personal project to build a task management application using React. This project focused on creating a dynamic, single-page application (SPA) with state management and persistent data storage using browser localStorage.

My Role & The Challenge

This was a self-directed project where I acted as both designer and developer. My goal was to deepen my understanding of React and create a tool I would genuinely use. The challenge was to implement features like adding, deleting, editing, and filtering tasks in a user-friendly interface.

My Process

I skipped extensive research as this was a personal tool but created simple wireframes and a component plan. The design prioritized a minimalist UI to keep the focus on the tasks themselves, using color-coding to indicate priority.

I built the application using React and the `useState` and `useEffect` hooks for state management. I implemented CRUD (Create, Read, Update, Delete) functionality for tasks and and used `localStorage` to persist data between sessions. Styled with Tailwind CSS for rapid UI development.

To demonstrate data visualization skills, I've added a sample chart showing task completion rates. This chart is built with Chart.js and is fully responsive.

Outcome & Learnings

The result is a functional SPA that I use daily. This project was instrumental in cementing my knowledge of React hooks and component lifecycle. It taught me a lot about state management and the challenges of creating a seamless user experience in a dynamic application.
